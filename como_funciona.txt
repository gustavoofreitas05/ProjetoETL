PIPELINE ETL — E-COMMERCE
Como funciona o projeto
=========================


O QUE É O PROJETO
------------------
Um pipeline ETL completo que busca dados de uma API pública de e-commerce,
transforma esses dados com Python e Pandas, e armazena tudo num banco SQLite
com queries analíticas prontas para gerar KPIs.

ETL = Extract (extrair) → Transform (transformar) → Load (carregar)


COMO RODAR
----------
1. python -m venv venv && source venv/bin/activate
2. pip install -r requirements.txt
3. cp .env.example .env
4. python src/pipeline.py


ESTRUTURA DE PASTAS
-------------------
src/
  pipeline.py              → ponto de entrada, orquestra as 3 fases
  extract/api_client.py    → fase 1: busca dados da API
  transform/transformer.py → fase 2: limpa e enriquece os dados
  load/loader.py           → fase 3: salva no SQLite e exporta CSVs

sql/
  analytics_queries.sql    → queries SQL prontas para análise

tests/
  test_extract.py          → testes da fase Extract
  test_transform.py        → testes da fase Transform
  test_load.py             → testes da fase Load

data/
  raw/                     → JSONs brutos salvos com timestamp
  processed/               → CSVs transformados
  ecommerce.db             → banco SQLite gerado pelo pipeline

logs/
  pipeline.log             → log completo de cada execução


FASE 1 — EXTRACT (api_client.py)
---------------------------------
Consome 4 endpoints da FakeStore API (fakestoreapi.com):
  - /products       → lista de produtos
  - /products/categories → categorias disponíveis
  - /carts          → carrinhos de compra (pedidos)
  - /users          → usuários

Funcionalidades:
  - Retry automático com backoff exponencial (espera 2s, 4s, 8s entre tentativas)
  - Salva o JSON bruto em data/raw/ com nome no formato raw_data_YYYYMMDD_HHMMSS.json
  - Logging de cada requisição (status, erros, tempo)


FASE 2 — TRANSFORM (transformer.py)
-------------------------------------
Recebe os dados brutos e aplica transformações com Pandas.

Nos produtos:
  - Remove duplicatas por ID
  - Separa o campo "rating" (que era um dicionário) em duas colunas: rating_rate e rating_count
  - Normaliza o texto das categorias (Title Case)
  - Converte preço de USD para BRL (cotação fixa de R$ 5,05 para demonstração)
  - Cria coluna rating_class: "alto" (>= 4.0), "médio" (>= 3.0), "baixo" (< 3.0)
  - Cria coluna revenue_potential: price × rating_count (proxy de popularidade)
  - Cria coluna price_range: "econômico" (< $25), "intermediário" (< $75), "premium" (< $200), "luxo"

Nos pedidos (carrinhos):
  - Desnormaliza os itens (cada produto do carrinho vira uma linha)
  - Faz join com a tabela de produtos para trazer preço e categoria
  - Calcula receita por item em USD e BRL

Também gera dois DataFrames de agregação:
  - category_summary: métricas agrupadas por categoria
  - top_products: os 5 produtos com maior receita potencial


FASE 3 — LOAD (loader.py)
--------------------------
Recebe os DataFrames transformados e:
  - Cria o schema no SQLite (tabelas: products, order_items, category_summary, top_products)
  - Carrega cada DataFrame na tabela correspondente
  - Exporta CSVs em data/processed/ para fácil acesso
  - Executa 5 queries analíticas e exporta os resultados em data/processed/analytics/

Queries analíticas geradas:
  1. receita_por_categoria      → receita total e média por categoria
  2. top5_produtos_receita      → ranking dos 5 melhores produtos
  3. distribuicao_faixa_preco   → quantidade e preço médio por faixa
  4. analise_avaliacoes         → métricas agrupadas por classe de avaliação
  5. pedidos_por_categoria      → volume e receita dos pedidos por categoria


ARQUIVO SQL (analytics_queries.sql)
-------------------------------------
Contém 8 queries prontas para rodar direto no banco ecommerce.db:
  1. Receita potencial por categoria
  2. Top 5 produtos por receita
  3. Distribuição por faixa de preço
  4. Análise de avaliações
  5. Pedidos por categoria
  6. Pedidos por usuário
  7. Produtos sem pedidos (LEFT JOIN)
  8. Correlação entre preço e avaliação por categoria (Pearson em SQL puro)


TESTES
------
Cada fase tem seu arquivo de testes unitários.
Para rodar: pytest tests/ -v

Os testes cobrem:
  - Retry na falha de conexão (Extract)
  - Remoção de duplicatas e colunas derivadas (Transform)
  - Criação de schema, carga e queries analíticas (Load)
  - Exportação de CSVs


TECNOLOGIAS
-----------
Python 3.10+     linguagem principal
Pandas           transformações e agregações
Requests         consumo da API REST
SQLite3          banco de dados (sem servidor, arquivo local)
python-dotenv    variáveis de ambiente via .env
pytest           testes unitários
logging          rastreamento de execução com arquivo de log
